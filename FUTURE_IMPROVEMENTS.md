# 향후 개선사항 (Future Improvements)

이 문서는 현재 즉시 구현하기 어렵지만, 향후 개선이 필요한 항목들을 정리한 목록입니다.

---

## 변호사 칼럼 (Blog) 개선사항

### 🟡 중간 우선순위

#### 1. 읽는 시간 표시
- **설명**: 각 칼럼의 예상 읽는 시간을 표시 (예: "5분 읽기")
- **구현 방법**:
  - 노션 페이지 본문의 글자 수 기반 계산
  - 평균 읽기 속도: 분당 200-250 단어(한글 기준)
  ```typescript
  const readingTime = Math.ceil((post.content?.length || 0) / 1000);
  ```
- **필요 작업**:
  - 상세 페이지 조회 시 콘텐츠 길이 계산
  - 목록 페이지에서는 노션에 별도 속성 추가 또는 캐싱 필요
- **예상 소요 시간**: 2-3시간
- **효과**: 사용자가 글 읽기 전 시간 투자 판단 가능

---

#### 2. 관련 칼럼 추천
- **설명**: 칼럼 상세 페이지 하단에 같은 카테고리/태그의 다른 칼럼 추천
- **구현 방법**:
  ```typescript
  // 같은 카테고리 또는 태그가 겹치는 칼럼 찾기
  const relatedPosts = allPosts
    .filter(p =>
      p.id !== currentPost.id &&
      (p.categories.some(cat => currentPost.categories.includes(cat)) ||
       p.tags.some(tag => currentPost.tags.includes(tag)))
    )
    .slice(0, 3);
  ```
- **필요 작업**:
  - BlogDetailClient에 관련 칼럼 섹션 추가
  - 서버에서 관련 칼럼 가져오는 API 또는 함수 추가
- **예상 소요 시간**: 3-4시간
- **효과**: 체류 시간 증가, 페이지 뷰 증가

---

#### 3. 조회수 자동 증가 로직
- **설명**: 칼럼 상세 페이지 조회 시 조회수 자동 증가
- **구현 방법**:
  - 옵션 1: API Route로 조회수 증가 엔드포인트 생성
    ```typescript
    // app/api/blog/[slug]/increment-view/route.ts
    export async function POST(request: Request, { params }: { params: { slug: string } }) {
      // 노션 페이지의 조회수 속성 +1
      await notion.pages.update({
        page_id: pageId,
        properties: {
          조회수: { number: currentViews + 1 }
        }
      });
    }
    ```
  - 옵션 2: Server Action 사용 (Next.js 14+)
  - 옵션 3: 외부 조회수 추적 서비스 (Google Analytics, Vercel Analytics)
- **필요 작업**:
  - IP 기반 중복 방지 로직 (쿠키 또는 세션)
  - 봇 트래픽 필터링
  - 실시간 반영 또는 배치 업데이트 결정
- **예상 소요 시간**: 4-6시간
- **효과**: 인기 글 파악, 데이터 기반 콘텐츠 전략

---

### 🟢 낮은 우선순위

#### 4. 태그 클릭 필터링
- **설명**: 태그 클릭 시 해당 태그가 포함된 칼럼만 필터링
- **구현 방법**:
  ```typescript
  const [selectedTag, setSelectedTag] = useState<string | null>(null);

  // 태그 클릭 핸들러
  <span onClick={() => setSelectedTag(tag)}>#{tag}</span>
  ```
- **필요 작업**:
  - 태그 필터 상태 추가
  - 카테고리 필터와 태그 필터 동시 적용 로직
  - 활성 태그 시각적 표시
- **예상 소요 시간**: 2-3시간
- **효과**: 세밀한 콘텐츠 탐색 가능

---

#### 5. 페이지네이션 또는 무한 스크롤
- **설명**: 칼럼이 많아질 경우 페이지네이션 또는 무한 스크롤 구현
- **구현 시기**: 칼럼 30개 이상부터 고려
- **구현 방법**:
  - 옵션 1: 페이지네이션 (1페이지당 12개)
  - 옵션 2: 무한 스크롤 (Intersection Observer API)
- **필요 작업**:
  - 노션 API 페이지네이션 처리 (start_cursor)
  - 클라이언트 상태 관리
- **예상 소요 시간**: 4-5시간
- **효과**: 성능 개선, 로딩 속도 향상

---

#### 6. 썸네일 이미지 지원
- **설명**: 각 칼럼에 썸네일 이미지 추가 (선택적)
- **구현 방법**:
  - 노션에 "썸네일" 파일 속성 추가
  - 없으면 기본 이미지 또는 카테고리별 이미지 사용
  - Next.js Image 컴포넌트로 최적화
- **필요 작업**:
  - 이미지 업로드 및 관리
  - 반응형 이미지 크기 조정
- **예상 소요 시간**: 3-4시간
- **효과**: 시각적 매력 증가, 클릭률 향상

---

#### 7. 칼럼 작성일 vs 발행일 구분
- **설명**: 작성일과 발행일을 분리하여 예약 발행 기능 지원
- **구현 방법**:
  - 노션에 "발행일" 속성 추가
  - 발행일이 현재보다 미래면 비공개 처리
  ```typescript
  const isPublished = post.published && new Date(post.publishedAt) <= new Date();
  ```
- **필요 작업**:
  - 노션 속성 추가
  - 필터 로직 수정
- **예상 소요 시간**: 1-2시간
- **효과**: 콘텐츠 스케줄 관리

---

#### 8. 칼럼 공유 시 Open Graph 이미지 자동 생성
- **설명**: SNS 공유 시 동적으로 Open Graph 이미지 생성
- **구현 방법**:
  - Next.js Image Generation API 사용
  - Vercel OG Image 라이브러리
- **필요 작업**:
  - 템플릿 디자인
  - API Route 구현
- **예상 소요 시간**: 4-6시간
- **효과**: SNS 공유 시 클릭률 향상

## 성공사례 (Cases) 개선사항

### 🟢 낮은 우선순위

#### 1. 사례 상세 페이지 개선
- **설명**: 현재 기본 마크다운 렌더링만 되어 있음
- **필요 개선**:
  - 타임라인 형식 (배경 → 전략 → 결과)
  - 핵심 수치 강조 (위자료 금액, 재산분할 비율 등)
  - 비포/애프터 비교 섹션
- **예상 소요 시간**: 6-8시간
- **효과**: 전문성 강화, 의뢰 전환율 향상

---

#### 2. 카테고리별 필터 + 정렬 조합
- **설명**: 카테고리 필터와 정렬을 동시에 사용 시 UX 개선
- **현재 문제**: 필터와 정렬이 독립적으로 작동
- **개선 방향**:
  - "위자료 + 금액 높은순" 같은 조합 가능
  - 필터 초기화 버튼 추가
- **예상 소요 시간**: 2-3시간

---

#### 3. 사례 북마크 기능
- **설명**: 관심 있는 사례를 저장하는 기능
- **구현 방법**: 로컬 스토리지 또는 사용자 계정 연동
- **예상 소요 시간**: 4-5시간
- **효과**: 재방문율 증가
