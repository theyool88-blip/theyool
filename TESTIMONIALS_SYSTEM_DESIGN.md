# 법무법인 더율 의뢰인 후기 시스템 설계서

## 1. 최종 선택: 하이브리드 방식 (Option 3 + 4)

### 핵심 전략
**"증빙이 곧 콘텐츠다"** - 사진 자체를 메인 콘텐츠로 활용하되, 클릭 시 풍부한 스토리텔링 제공

### 추천 이유 (평가 점수)
- **신뢰도**: 9/10 - 조작 불가능한 실제 증빙
- **감성 연결**: 9/10 - 진짜 의뢰인의 생생한 목소리
- **가독성**: 8/10 - 모바일 최적화, 단계적 정보 제공
- **차별화**: 10/10 - 업계 최초 증빙 중심 후기
- **전환율**: 9/10 - 신뢰 구축 → 상담 유도 자연스러운 흐름

## 3. 사진 유형별 처리 방법

### 3.1 카카오톡
- **표시 방식**: 대화 흐름 3-4개 연속 캡처
- **블러 처리**: 프로필 사진, 이름 (대화 내용은 유지)
- **강조 포인트**: 이모티콘, 감사 표현, 금액 언급
- **메타데이터**: "카카오톡 | [카테고리] | [날짜]"

### 3.2 문자 메시지
- **표시 방식**: 단일 메시지 확대
- **블러 처리**: 전화번호 뒷자리
- **강조 포인트**: 짧고 강렬한 핵심 메시지
- **메타데이터**: "문자 | [카테고리] | [날짜]"

### 3.3 네이버 지도 리뷰
- **표시 방식**: 별점 + 리뷰 전문
- **블러 처리**: 프로필 이름 일부
- **강조 포인트**: 5점 만점, 공식 플랫폼 인증
- **메타데이터**: "네이버 리뷰 | ★★★★★ | [날짜]"

### 3.4 손편지
- **표시 방식**: 편지 전체 + 일부 텍스트 추출
- **블러 처리**: 개인정보 부분만
- **강조 포인트**: 손글씨 감성, 정성
- **메타데이터**: "손편지 | [카테고리] | [날짜]"

## 4. 성공사례와의 연계

### 4.1 연결 구조
```
성공사례 상세 페이지
├── 사건 개요
├── 진행 과정
├── 승소 포인트
└── 📌 실제 의뢰인 후기 (NEW!)
    └── [관련 후기 2-3개 자동 매칭]
```

### 4.2 매칭 로직
- 동일 카테고리 (위자료, 재산분할 등)
- 유사 금액대
- 비슷한 시기

## 5. 데이터베이스 구조

```sql
-- testimonials 테이블
CREATE TABLE testimonials (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),

  -- 기본 정보
  category TEXT NOT NULL, -- 'alimony', 'property', 'custody', 'adultery'
  case_type TEXT NOT NULL, -- '위자료', '재산분할', '양육권', '상간'
  amount BIGINT, -- 금액 (원 단위, nullable)

  -- 증빙 정보
  evidence_type TEXT NOT NULL, -- 'kakao', 'sms', 'naver', 'letter'
  evidence_images JSONB NOT NULL, -- [{url, order, caption}]
  evidence_date DATE, -- 후기 작성 날짜

  -- 의뢰인 정보 (익명화)
  client_age_group TEXT, -- '30대', '40대', '50대'
  client_gender TEXT, -- 'male', 'female'
  court_location TEXT, -- '서울가정법원', '강남지원' 등

  -- 콘텐츠
  highlight_text TEXT, -- 한 줄 하이라이트 (카드에 표시)
  full_story TEXT, -- 전체 스토리 (라이트박스에서 표시)
  key_points JSONB, -- ['2억 승소', '6개월 소송', '숨긴 재산 발견']

  -- 관계
  related_case_id UUID REFERENCES cases(id), -- 연결된 성공사례

  -- 메타데이터
  display_order INTEGER, -- 노출 순서
  is_featured BOOLEAN DEFAULT FALSE, -- 메인 노출 여부
  is_published BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_testimonials_category ON testimonials(category);
CREATE INDEX idx_testimonials_evidence_type ON testimonials(evidence_type);
CREATE INDEX idx_testimonials_featured ON testimonials(is_featured, display_order);
```

## 6. 모바일/데스크톱 차이점

### 모바일 (Mobile First)
- 전체 너비 활용 이미지
- 스와이프 제스처 중심
- 단계적 정보 공개 (탭하여 더보기)
- 하단 고정 CTA 버튼

### 데스크톱
- 호버 시 간단 정보 표시
- 클릭 시 모달 팝업
- 좌우 화살표 네비게이션
- 여유 있는 여백과 타이포그래피

## 7. 구현 우선순위

1. **Phase 1**: 기본 증빙 사진 업로드 및 표시
2. **Phase 2**: 라이트박스 및 스토리 기능
3. **Phase 3**: 성공사례 연동
4. **Phase 4**: 필터 및 검색 기능

## 8. 성공 지표 (KPI)

- 페이지 체류 시간 30% 증가
- 후기 → 상담 신청 전환율 15% 이상
- 증빙 사진 클릭률 50% 이상
- 평균 3개 이상 후기 열람

---

이 시스템은 **"진정성"**을 핵심 가치로, **증빙의 힘**을 최대한 활용하여 잠재 의뢰인에게 **신뢰와 희망**을 동시에 전달합니다.
