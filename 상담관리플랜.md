## 1. 전략 개요

### 1.1 비즈니스 목표

**상담 우선순위 (전환율 기준)**
```
👑 방문상담 (Visit) ─────────→ 계약 체결 (HIGHEST VALUE)
    ↑
    │ 전환 유도
    │
⚡ 전화상담 (Phone, 10분 무료) → 방문 유도 게이트웨이
    ↑
    │ 보조 채널
    │
📹 화상상담 (Video) ─────────→ 거리/시간 제약 시 대안
💬 카카오톡 (KakaoTalk) ─────→ 지원 채널 (낮은 전환율)
```

**핵심 인사이트**
- 10분 전화상담의 진정한 목적: 신뢰 구축 → 방문 예약 유도
- 방문 상담이 계약 전환의 결정적 순간
- 사용자 여정: 인지 → 전화 → 신뢰 → 방문 → 계약

### 1.2 사용자 여정 맵

```
┌─────────────────────────────────────────────────────────────────┐
│                     USER JOURNEY TO CONTRACT                     │
└─────────────────────────────────────────────────────────────────┘

1. 인지 단계 (Awareness)
   - 검색/SNS/추천 → 홈페이지 방문
   - 성공사례/칼럼/FAQ 탐색
   - 목표: 신뢰 형성 + 상담 결심

2. 첫 접촉 (First Touch) ⚡
   - 전화 10분 무료상담 (즉시)
   - 목표: 빠른 문제 파악 + 방문 예약 제안

3. 심화 상담 (Deep Consultation) 👑
   - 방문 상담 (천안/평택 사무소)
   - 또는 화상 상담 (불가피한 경우)
   - 목표: 전략 제시 + 신뢰 완성

4. 계약 결정 (Decision) 🎯
   - 당일 계약 또는 고민 후 재방문
   - 목표: 계약 체결

5. 사후 관리 (Retention)
   - 진행 상황 공유
   - 추가 상담 요청 대응
```

### 1.3 성공 지표 (KPI)

| 지표 | 현재 | 목표 (6개월 후) |
|------|------|----------------|
| 전화 → 방문 전환율 | 측정 필요 | 35% |
| 방문 → 계약 전환율 | 측정 필요 | 65% |
| 화상 → 계약 전환율 | 측정 필요 | 40% |
| 평균 응답 시간 | 30초 (추정) | 20초 유지 |
| 예약 노쇼율 | 측정 필요 | 15% 이하 |
| 월 방문 상담 건수 | 측정 필요 | 50건 |

## 4. 상담 방식별 상세 설계

### 4.1 방문 상담 예약 시스템 👑 (최우선)

#### User Flow
```
[홈페이지]
  → "방문 상담 예약" 버튼 클릭
  → [BookingForm - Step 1] 상담 유형 확인
  → [BookingForm - Step 2] 사무소 선택 (천안/평택)
  → [BookingForm - Step 3] 날짜 선택 (캘린더)
  → [BookingForm - Step 4] 시간 선택 (09:00-18:00, 1h 단위)
  → [BookingForm - Step 5] 개인정보 (이름/전화/이메일/상담분야/상황)
  → [BookingForm - Step 6] 확인 및 제출
  → [확인 페이지] 예약 완료 + 준비물 + 찾아오는 길
  → (선택) 관리자 확정 → SMS/이메일 발송
```

#### 필수 폼 필드
| 필드 | 필수 여부 | 용도 |
|------|----------|------|
| 상담 유형 | ✅ | visit (고정) |
| 사무소 | ✅ | cheonan / pyeongtaek |
| 날짜 | ✅ | 예약일 (YYYY-MM-DD) |
| 시간 | ✅ | 09:00, 10:00, ..., 17:00 |
| 이름 | ✅ | 본인 확인 |
| 전화번호 | ✅ | 연락처 |
| 이메일 | ❌ | 확인 메일 발송 |
| 상담 분야 | ❌ | 위자료/재산분할/양육권 |
| 상황 설명 | ❌ | 사전 준비 |

#### 예약 확정 프로세스
1. **자동 확정 (권장)**
   - 예약 즉시 "확정" 상태로 저장
   - 고객에게 확인 메시지 표시
   - 관리자는 필요시 취소 가능

2. **수동 확정 (대안)**
   - 예약 시 "대기" 상태
   - 관리자 검토 후 확정 버튼
   - 확정 시 고객에게 SMS/이메일 발송

#### 준비물 안내 (확인 페이지)
```
📋 방문 상담 준비물

필수 서류
  - 혼인관계증명서
  - 가족관계증명서
  - 주민등록등본

있으면 좋은 것
  - 재산 목록 (부동산, 예금, 주식 등)
  - 증거 자료 (문자, 사진, 녹음)
  - 상대방 정보 (이름, 연락처)
  - 상황 정리 메모

💡 없어도 상담은 가능합니다
   상황만 말씀해주시면 필요한 것을 안내해드립니다
```

#### 찾아오는 길
- 사무소별 주소, 지도, 주차 안내
- 대중교통 안내
- 사무소 사진 (선택)

#### Database Schema
```sql
CREATE TABLE bookings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  type TEXT NOT NULL CHECK (type IN ('visit', 'video')),
  office TEXT CHECK (office IN ('cheonan', 'pyeongtaek')),
  booking_date DATE NOT NULL,
  time_slot TIME NOT NULL,
  name TEXT NOT NULL,
  phone TEXT NOT NULL,
  email TEXT,
  category TEXT,
  message TEXT,
  status TEXT NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'confirmed', 'cancelled', 'completed')),
  video_link TEXT,
  admin_notes TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(office, booking_date, time_slot) -- 중복 예약 방지
);

CREATE INDEX idx_bookings_date ON bookings(booking_date);
CREATE INDEX idx_bookings_status ON bookings(status);
```

#### Admin Interface
- **예약 목록 테이블**
  - 날짜/시간, 고객명, 전화, 사무소, 상태
  - 필터: 날짜 범위, 사무소, 상태
  - 검색: 이름, 전화번호

- **예약 상세 모달**
  - 모든 예약 정보 표시
  - 액션: 확정, 취소, 메모 추가
  - (선택) SMS/이메일 재발송

### 4.2 화상 상담 예약 시스템 📹

#### User Flow
```
[홈페이지]
  → "화상 상담 예약" 버튼 클릭
  → [BookingForm - Step 1] 상담 유형 확인 (video)
  → [BookingForm - Step 2] 날짜 선택
  → [BookingForm - Step 3] 시간 선택
  → [BookingForm - Step 4] 개인정보 입력
  → [BookingForm - Step 5] 확인 및 제출
  → [확인 페이지] 예약 완료 + 화상 링크 안내
  → 관리자가 Zoom/Meet 링크 생성 → 고객에게 전송
```

#### 화상 플랫폼 선택
**옵션 A: Zoom (권장)**
- Zoom API로 미팅 자동 생성
- 장점: 안정적, 녹화 가능
- 단점: API 설정 필요

**옵션 B: Google Meet (간편)**
- Google Calendar API로 이벤트 생성 시 Meet 자동 생성
- 장점: 무료, 간편
- 단점: Google 계정 필요

**옵션 C: 수동 입력 (MVP)**
- 관리자가 수동으로 링크 생성 후 입력
- 장점: 구현 빠름
- 단점: 수동 작업

**결론**: Phase 1은 옵션 C (수동), Phase 2에서 API 자동화

#### 화상 상담 특이사항
- 사무소 선택 불필요
- 기술 요구사항 안내 (카메라, 마이크, 인터넷)
- 화상 링크는 예약 확정 후 제공

#### 확인 페이지 내용
```
✅ 화상 상담 예약 완료

📅 일시: 2025년 11월 25일 (월) 14:00
💻 플랫폼: Zoom

화상 링크는 상담 1일 전에 문자로 보내드립니다.

📱 기술 준비사항
  - 웹캠 (카메라)
  - 마이크
  - 안정적인 인터넷 연결
  - Zoom 앱 설치 (선택, 웹 브라우저도 가능)

📋 준비물
  - 상담 받고 싶은 내용 정리
  - 관련 서류 (화면 공유 가능)
```

### 4.3 전화번호 남기기 (콜백) 📞

```tsx
interface CallbackFormData {

```

- **옵션 B**: 특정 랜딩 페이지에만 사용 (상황 선택이 유의미한 경우)

## 8. 기술 고려사항

### 8.1 캘린더 라이브러리 선택

**Option 1: react-calendar**
- 장점: 가볍고 커스터마이징 쉬움
- 단점: 시간 선택 UI 별도 구현 필요
- 추천도: ⭐⭐⭐⭐

**Option 2: @fullcalendar/react**
- 장점: 풀 기능, 관리자 대시보드에도 사용 가능
- 단점: 무거움, 오버킬
- 추천도: ⭐⭐⭐

**Option 3: react-datepicker**
- 장점: 날짜 + 시간 통합, 인기 많음
- 단점: 디자인 커스터마이징 제한적
- 추천도: ⭐⭐⭐⭐⭐ (권장)

**결론**: react-datepicker 사용

### 8.2 예약 충돌 방지

**방법 1: Database Unique Constraint**
```sql
UNIQUE(office, booking_date, time_slot)
```
- 장점: 100% 안전
- 단점: 에러 처리 필요

**방법 2: Optimistic Locking**
- `updated_at` 필드 비교
- 동시 수정 시 에러 반환

**방법 3: Redis Lock (고급)**
- 예약 시도 시 임시 락
- 3분 타임아웃

**권장**: 방법 1 (Unique Constraint) + 방법 2 (Optimistic Locking)

### 8.3 시간대 처리

- 모든 시간은 **한국 시간 (KST, UTC+9)** 기준
- Database에는 `TIME` 타입 또는 `TEXT` (HH:MM) 저장
- 날짜는 `DATE` 타입 (YYYY-MM-DD)

### 8.4 화상 링크 생성

**Phase 1 (MVP): 수동 입력**
- 관리자가 Zoom/Meet에서 수동으로 미팅 생성
- 링크를 Admin UI에 붙여넣기

**Phase 2 (자동화): Zoom API**
```typescript
// Example: Zoom Meeting Creation
const createZoomMeeting = async (booking: Booking) => {
  const response = await fetch('https://api.zoom.us/v2/users/me/meetings', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${ZOOM_JWT_TOKEN}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      topic: `법무법인 더율 상담 - ${booking.name}`,
      type: 2, // Scheduled meeting
      start_time: `${booking.booking_date}T${booking.time_slot}:00`,
      duration: 60,
      timezone: 'Asia/Seoul',
    }),
  });

  const data = await response.json();
  return data.join_url;
};
```
