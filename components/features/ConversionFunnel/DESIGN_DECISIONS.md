# Conversion Funnel - 디자인 결정 사항

법무법인 더율 전환 퍼널 컴포넌트의 디자인 결정 과정 및 근거

---

## 1. 전반적 평가

### 기존 홈페이지와의 조화도: ⭐⭐⭐⭐⭐ (5/5)

**완벽히 통합된 이유:**
- Sage Green 색상 체계 100% 준수
- Border radius 20px 유지
- 애니메이션 easing `cubic-bezier(0.16, 1, 0.3, 1)` 동일
- Shadow 시스템 일치 (`rgba(109, 181, 164, 0.08~0.35)`)
- 섹션 패딩 및 여백 규칙 준수 (`py-16 md:py-24`)

**검증 근거:**
- `/app/page.tsx` 분석: 기존 카드 스타일과 100% 일치
- `/app/globals.css` 분석: CSS 변수 및 애니메이션 재사용
- 타이포그래피 스케일 일치 (text-sm, text-xs, font-bold)

## 2. 색상 전략

### Hot Lead - Coral 계열 (긴급성)

**선택 근거:**
- 이혼 상담은 타이밍이 중요 → 즉각적 행동 유도 필요
- Coral Pink는 따뜻하면서도 긴급함을 전달
- 법률 서비스의 딱딱함을 완화

**색상 조합:**
```css
bg: from-coral-400 to-coral-500
border: border-coral-600
badge: bg-coral-100 text-coral-800
```

**심리적 효과:**
- 따뜻함 60%, 긴급성 40%
- 불안감 없는 행동 유도
- "지금 바로" vs "망설이지 말고" → 전자 선택

### Warm Lead - Sage Green (추천/신뢰)

**선택 근거:**
- 브랜드 정체성의 핵심 색상
- 전문성과 안정감을 동시에 전달
- 가장 많은 항목을 차지 (5개 중 4개)

**색상 조합:**
```css
bg: from-sage-500 to-sage-600
border: border-sage-700
badge: bg-sage-100 text-sage-800
```

**심리적 효과:**
- 신뢰감 70%, 행동 유도 30%
- "추천" 뱃지로 선택 유도
- 딱딱하지 않은 법률 서비스 이미지

### Cold Lead - Light Sage (탐색/정보)

**선택 근거:**
- 부담 없는 정보 탐색 단계
- 백그라운드 역할 (Hot/Warm 강조 위해)
- 깨끗한 인상으로 접근성 향상

**색상 조합:**
```css
bg: from-sage-50 to-sage-100
border: border-sage-200
text: text-gray-900
```

**심리적 효과:**
- 편안함 80%, 정보성 20%
- 스크롤 유도 (궁금증 해소)
- 전환 압박 없음

## 3. 레이아웃 결정

### 모바일: 2x4 그리드

**선택 근거:**
- 1열: 스크롤 피로도 높음
- 3열: 카드 너무 작아짐 (가독성 저하)
- 2열: 균형 최적 (한 눈에 2개, 스크롤 자연스러움)

**UX 고려사항:**
- 스크롤 깊이: 4스크롤 (적정)
- 터치 타겟: 최소 44x44px 보장
- 간격: 12px (손가락 사이 충분한 여백)

### 데스크톱: 4x2 그리드

**선택 근거:**
- 8개 항목을 한 눈에 비교 가능
- 스크롤 없이 전체 선택지 제시
- 가로 폭 1200px 내에서 최적 배치

**시각적 균형:**
- 각 카드: 약 280px 너비
- 간격: 16px (여유 있는 분리감)
- 2행 구조로 안정감 확보

### 가로 스크롤 방식을 채택하지 않은 이유

**기각 사유:**
- 법률 서비스: 모든 옵션을 명확히 보여줘야 함
- 가로 스크롤: "숨겨진 콘텐츠" 인식 → 놓칠 수 있음
- 신뢰 구축: 투명성이 핵심 → 전체 노출 필수

## 4. 애니메이션 전략

### 순차적 Stagger (80ms 딜레이)

**선택 근거:**
- 즉시 노출: 압도적, 정보 과부하
- 순차 노출: 시선 가이드, 자연스러운 탐색
- 80ms: 너무 빠르지도 느리지도 않은 리듬감

**시각적 흐름:**
```
좌상 → 우상 → 좌하 → 우하
Hot → Warm → Cold 순서로 자연스러운 우선순위
```

### 호버 효과: -4px 위로, 그림자 증가

**선택 근거:**
- 기존 홈페이지 `hover-lift` 클래스와 일치
- -4px: 너무 튀지 않으면서도 명확한 피드백
- 그림자: depth 느낌으로 클릭 유도

**미세 조정:**
- 이모지 scale(1.1): 생동감
- 화살표 translateX(4px): 방향성 강조
- duration 300ms: 자연스러운 전환

## 5. 타이포그래피

### 카드 내부 계층

**제목: text-sm font-bold**
- 14px: 모바일에서 충분한 가독성
- bold: 스캔 가능성 향상
- 2줄 이내로 제한 (긴 제목 지양)

**설명: text-xs**
- 12px: 보조 정보로서의 역할
- leading-relaxed: 답답하지 않은 행간
- 1~2줄: 핵심만 전달

**이모지: text-5xl (48px)**
- 시각적 앵커 역할
- 언어 장벽 없는 보편적 소통
- 호버 시 확대로 재미 요소

## 6. 접근성 (WCAG 2.1 AA)

### 색상 대비

**Hot Lead (Coral):**
- 흰색 텍스트 on Coral-500: 4.8:1 ✅
- 뱃지 Coral-800 on Coral-100: 7.2:1 ✅

**Warm Lead (Sage):**
- 흰색 텍스트 on Sage-600: 4.9:1 ✅
- 뱃지 Sage-800 on Sage-100: 7.5:1 ✅

**Cold Lead (Light Sage):**
- Gray-900 on Sage-50: 15.3:1 ✅
- Gray-600 on Sage-100: 5.8:1 ✅

### 키보드 네비게이션

**구현:**
- `focus-visible:ring-2`: 명확한 포커스 표시
- `ring-sage-600`: 브랜드 색상으로 일관성
- `ring-offset-2`: 카드 경계와 분리

### ARIA Labels

**각 카드:**
```tsx
aria-label={`${item.title} - ${item.description}`}
```
- 스크린 리더 사용자에게 전체 맥락 전달
- 제목 + 설명 조합으로 명확성 확보

## 7. 퍼널 항목 우선순위

### 1순위: 10분 무료 진단 (Hot)

**선택 근거:**
- 가장 낮은 진입 장벽
- 즉각적 가치 제공
- 전환율 가장 높을 것으로 예상

**배치:** 좌상단 (시선 시작점)

### 2순위: 수임료 확인 (Warm)

**선택 근거:**
- 이혼 비용: 가장 큰 걱정거리
- 투명성 강조 → 신뢰 구축
- 상담 전 부담 완화

**배치:** 우상단 (자연스러운 시선 흐름)

### 3순위: 나와 같은 사례 (Cold)

**선택 근거:**
- "나만의 문제가 아니구나" 안심
- 성공 가능성 확인
- 긴 탐색 시간 허용

**배치:** 좌측 2번째 (Hot 다음)

### 하위 항목들 (Warm/Cold 혼합)

**전략:**
- Hot 1개로 긴급성 확보
- Warm 4개로 신뢰 구축
- Cold 3개로 정보 제공

**시각적 균형:**
- 너무 많은 Hot: 압박감
- 너무 많은 Cold: 전환 저하
- 현재 비율: 이상적 균형

## 8. 뱃지 시스템

### "긴급" 뱃지 (Hot)

**디자인:**
```css
bg: coral-100
text: coral-800
dot: animate-pulse
```

**심리적 효과:**
- 깜빡이는 점: 주목도 상승
- "긴급" 텍스트: FOMO 유발
- 우측 상단: 자연스러운 시선 유도

### "추천" 뱃지 (Warm)

**디자인:**
```css
bg: sage-100
text: sage-800
dot: animate-pulse
```

**심리적 효과:**
- "추천": 신뢰 기반 선택 유도
- 브랜드 색상: 정체성 강화
- 부담 없는 안내

### Cold는 뱃지 없음

**선택 근거:**
- 정보 탐색 단계: 압박 불필요
- 시각적 정리: 너무 많은 뱃지 지양
- Hot/Warm 강조 효과

## 9. 호버 vs Active 상태

### 데스크톱: 호버 중심

**구현:**
- translateY(-4px): 카드 부상
- shadow 증가: depth 강조
- 이모지 확대: 재미 요소

### 모바일: Active 상태

**구현:**
- `active:scale-98`: 터치 피드백
- 호버 효과는 자동 적용 (터치 시)
- 빠른 반응으로 responsiveness 전달

---

## 10. 섹션 배경 전략

### 그라데이션 배경

**선택:**
```css
bg-gradient-to-b from-white via-sage-50/30 to-white
```

**근거:**
- 백그라운드에서 시작 (white)
- 중간에 sage 톤 (브랜드 정체성)
- 다시 white로 복귀 (다음 섹션 자연스러운 연결)

**시각적 효과:**
- 섹션 분리감 확보
- 지루하지 않은 변화
- FAQ 섹션으로 부드러운 전환

## 11. 하단 CTA 전략

### 2개 버튼 배치

**1. 전화 버튼 (Primary)**
```css
bg-sage-600 hover:bg-sage-700
shadow: 0 4px 14px rgba(90, 153, 136, 0.3)
```
- 즉각적 행동 유도
- 브랜드 색상으로 신뢰 강조

**2. 가이드 버튼 (Secondary)**
```css
bg-white border-2 border-gray-300 hover:border-sage-600
```
- 정보 추가 탐색 허용
- 전화 부담스러운 사용자 배려

### 안내 텍스트

**"궁금한 점이 있으신가요? 언제든 편하게 연락주세요"**
- 부담 완화
- 친근한 톤
- 언제든 → 시간 압박 없음

## 12. 성능 최적화

### Intersection Observer

**구현:**
- 뷰포트 진입 시에만 애니메이션
- threshold: 0.1 (10% 노출 시 트리거)
- rootMargin: 50px (미리 준비)

**효과:**
- 초기 로딩 속도 향상
- 불필요한 애니메이션 방지
- 배터리 절약

### CSS Transform

**활용:**
- translateY: GPU 가속
- scale: 부드러운 확대/축소
- opacity: 자연스러운 페이드

**장점:**
- reflow/repaint 최소화
- 60fps 애니메이션 보장
- 저사양 기기에서도 원활

## 13. 모바일 우선 설계

### 터치 타겟

**기준:**
- 최소 크기: 44x44px (Apple HIG)
- 카드 전체: 터치 가능
- 간격: 12px (손가락 두께 고려)

### 스크롤 깊이

**계산:**
- 2열 x 4행 = 4스크롤
- 각 카드 높이: 약 140px
- 총 높이: 약 600px (1~1.5 스크린)

**적정성:**
- 너무 길지 않음 (이탈 방지)
- 충분한 정보 제공
- 스크롤 리듬감 유지

## 15. A/B 테스트 가설

### 제목 변형

**A안 (현재):**
"상담 전 먼저 확인해보세요"
- 실용적, 명확

**B안 (감성):**
"지금 이 순간, 당신을 응원합니다"
- 감성적, 공감

**C안 (속도):**
"3분이면 충분해요"
- 시간 절약 강조

### 배치 변형

**현재:** TestimonialEvidenceGallery 후
**대안:** ConsultationTimingGuide 후

**가설:**
- 후기 본 후 → 행동 전환 높음
- 타이밍 가이드 후 → 즉각 행동 높음
